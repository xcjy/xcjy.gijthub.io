react hooks学习

useState

  1. 不可局部更新. 局部更新要用…

​ 2.地址要变, 否则改变不了

setState(obj) obj地址如果不变，那么react就认为数据没有变化

  1. useState 可接受函数

  2. setState 接受函数

    setN(i => i+1)

useRef

值在组件不断render时 保持不变。而useState是会变的。

const count=useRef(0)

读取count.current

当count.value 变化时 不会自动render vue3会自动render

与createRef的区别：

区别在于 createRef 将始终创建一个新的ref。在基于类的组件中，通常在构造期间将ref放在实例属性中（例如 this.input = createRef（））。您在功能组件中没有此选项。 useRef 负责每次返回与初始渲染时相同的ref。

forwardRef

由于props不包含 ref 所以需要forwardRef

useImperativeHandle

应该叫setRef

UseReducer

useReducer是useState的复杂版

一、创建初始值initialState

二、创建所有操作reducer(state,action)

三、 传给useReducer 得到读写api

​ const [n,dispatch]= useReducer(reducer,intialState)

四、 调用写api dispatch({type:’add’, payload:’xxx’})

useContext

上下文是局部的全局变量

一、 const xxx=createContext(initial)创建上下文

二、 <xxx.provider>来圈定作用域

三、 作用域内使用const [n,setN]=useContext(xxx) 来使用上下文

UseEffect

副作用：对环境的改变 如修改document.title console.log

不一定非要把副作用放在useEffect

用法：

UseLayoutEffect

useEffect浏览器渲染完成后执行

useLayoutEffect在浏览器渲染前执行

useLayoutEffect 的任务 最好影响了layout（外观）..

为了用户体验 优先使用useEffect （优先渲染）

useMemo,useCallback

React.memo 可以解决 多余的render（子组件props没变但它还是执行了） 。 如果props不变就不会再执行 但加了监听函数后还是会执行

useMemo 解决了这个问题，使函数能够重用.类似于vue的computed

第一个参数是()=>value ,

第二个参数是依赖[m,n] 只有依赖变化时 才会计算出新的value。。

useMemo ( ()=>(x)=>console.log(x) )

useCallback(x =>log(x),[m] ) 等价于 useMemo( ()=>x=>log(x),m)

数据是否要变化的hook 小结

数据是否要变化的hook,

  1. useState , useReducer n 每次变化
  2. useMemo /useCallback [m] 变时才改变
  3. useRef 永远不变。
